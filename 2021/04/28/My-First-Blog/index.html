开启一场属于你自己的Blog旅程吧！Github+Hexo搭建个人网站心酸记录史:cry:

前言

终于！我也有自己的博客网站了！嘻嘻嘻~ 但是搭建过程真的太难了叭！记录一下自己搭建博客的心酸历程，可以帮助到像我一样的小白同志就更加美滋滋了！我搭建的博客没有利用属于自己的域名哦！（只想白嫖）后期可能会搭建属于自己的个人网站！让我们一起走进自己的博客世界叭！

前期准备

Hexo是基于Node.js的静态博客框架，它是一个简单、快速、强大的基于Github的博客发布工具，支持Markdown格式，有众多优秀插件和主题。
前期我们需要下载安装Git、Node.js以及Github上创建自己的仓库。

下载安装Git

Git是开源的分布式版本控制系统，用于敏捷高效地处理项目。我们在本地上搭建好个人博客后，需要使用Git同步到GitHub上。从Git官网上根据自己电脑系统下载相关安装包进行安装，安装完毕后，可以通过命令窗口输入git --version测试是否安装成功，成功界面如下：

下载安装Node.js

Hexo基于Node.js，因此我们需要在Node.js官网下载相关安装包，注意安装Node.js会包含环境变量及npm的安装，安装完毕后可以通过在命令窗口输入node -v测试Node.js是否安装成功，成功界面如下：

检测npm是否安装成功，输入npm -v，安装成功界面如下：

GitHub创建仓库，配置SSH key

  1. 创建个人仓库

登录Github账号（如果事先没有账号，可以用邮箱注册一个账号），点击Github中的New Repositorise创建新仓库：

注意创建的仓库名必须严格按照以下格式，**username**.github.io（注意这里的username必须与你的Github注册的用户名一致，否则无效），因此一个Github账户最多只能创建一个这样可以直接使用域名访问的仓库：

  1. 配置SSH Key
    我们需要将我们本地创建的博客发布到Github上，就需要通过Git与Github账号绑定，但是Github提交代码必须要拥有权限，直接使用账号和密码不太安全，并且每次提交都要输入账号密码，相对繁琐，因此可以通过Git配置SSH Key解决本地和服务器的连接问题。

  • 菜单搜索Git Bash，鼠标右击打开Git Bash：

  • 在Git Bash中设置Github账号的用户名和注册邮箱的信息，输入命令如下：
    git config --global user.name 这里面的user.name是你的Github用户名；
    git config --global user.email这里面的user.email是你的Github注册邮箱；

  • 然后，生成SSH密钥文件，输入命令：ssh-keygen -t rsa -C user.email这里面user.email是你的Github注册邮箱；

  • 直接三个回车，默认不需要设置密码；

  • 找到生成的.ssh\id_rsa.pub文件，用记事本打开密钥，将内容全部复制；

  • 打开Github_Settings_keys页面，新建new SSH Key：


Title为标题，任意填即可，将刚刚复制的id_rsa.pub内容粘贴进去，最后点击Add SSH key；

  • 在Git Bash中检测GitHub公钥设置是否成功，输入命令ssh git@github.com，如显示下面界面则配置成功：


这里之所以设置GitHub密钥原因是，通过非对称加密的公钥与私钥来完成加密，公钥放置在GitHub上，私钥放置在自己的电脑里。GitHub要求每次推送代码都是合法用户，所以每次推送都需要输入账号密码验证推送用户是否是合法用户，为了省去每次输入密码的步骤，采用了ssh，当你推送的时候，git就会匹配你的私钥跟GitHub上面的公钥是否是配对的，若是匹配就认为你是合法用户，则允许推送。这样可以保证每次的推送都是正确合法的。

Hexo安装与使用

由于Github pages存放的都是静态文件，但是博客存放的不只是文章内容，还有文章列表、分类、标签、翻页等动态内容，假如每次写完一篇文章都要手动更新博文目录和相关链接信息，相信谁都会疯掉，所以Hexo所做的就是将这些.md文件都放在本地，每次写完文章后调用写好的命令来批量完成相关页面的生成，然后再将有改动的页面提交到Github上。

前期已经全部完成了Hexo的安装环境，接下来我们就要安装Hexo啦！

Hexo的安装

  1. Hexo就是我们的个人博客网站的框架， 这里需要自己在电脑里创建一个文件夹，可以命名为MyBlog，Hexo框架与以后你自己发布的网页都在这个文件夹中。创建好后，进入文件夹中，右击选择Git Bash Here，我们这里使用git bash完成Hexo的安装；

  2. 使用npm命令安装Hexo，输入命令npm install -g hexo-cli

  3. 安装完成后，进行初始化，输入命令hexo init，这时Hexo会自动下载相关文件夹在MyBlog目录中：

  1. 检测网站的初始化模样，可以输入以下命令：

    hexo new test_my_fristblog 新建博客文章；

    hexo g 生成网页；

    hexo s 开启本地预览服务；

  2. 此时，我们就可以打开浏览器访问http://localhost:400，看到默认主题下的博客网站。很多人会碰到浏览器一直在转圈但是就是加载不出来的问题，一般情况下是因为端口占用的缘故，因为4000这个端口太常见了，解决端口冲突问题请参考这篇文章：http://blog.liuxianan.com/windows-port-bind.html

Hexo的相关命令

npm install hexo -g 安装Hexo

npm update hexo -g 升级Hexo

hexo init 初始化博客

hexo n blog 新建博客文章blog

hexo g 生成网页

hexo s 开启本地服务预览

hexo d 更新部署

hexo clean 清除缓存

hexo server Hexo会监视文件变动并自动更新，无须重启服务器

hexo server -s 静态模式

hexo server -p 5000 更改端口

hexo server -i 192.168.1.1 自定义IP

Hexo站点配置文件的配置和网站的推送

我们上面只是本地预览，还没有实现发布网站，将我们的网站推送给更多的人访问。首先了解一下站点配置文件和主题配置文件，在我们创建的博客文件夹根目录下有一个_config.yml文件称为站点配置文件，如下图所示：

在根目录下的themes文件夹下，里面也有一个_config.yml文件，这个称为主题配置文件，用于设置主题相关样式，如图所示：

接下来，我们就要将Hexo与Github关联起来，首先我们需要下载安装Git部署插件，输入命令npm install hexo-deployer-git --save

部署插件安装完成后，打开我们的站点配置文件_config.yml，找到页面最后deploy: 进行相关配置：

deploy:
type: git
repo: Github上创建仓库的完整路径，可以去Github上复制，记得一定要加上.git
branch: master

Github复制完整的仓库路径：

配置文件参考如下：

保存配置文件，我们的配置就完成了，这时候输入hexo d部署网站命令，就可以打开浏览器，在地址栏输入你的放置个人网站的仓库路径，即username.github.io，你的个人博客就上线了，可以在网络上被访问了！！

总结推送网站的四条命令，前提是必须必须必须完成站点配置文件的相关配置：

hexo clean 清除缓存，一般浏览器不能实时访问，出现一些问题，先进行缓存清除；

hexo n blog 创建博客文章；

hexo g 生成网站；

hexo d 部署网站，将个人博客推送到Github上；

Hexo主题

如果你不喜欢Hexo默认主题，可以更换不同的主题，Hexo提供了各种各样的主题风格，借下来我就一起看看如何切换自己喜欢的主题风格叭！

打开主题任意门，如图可以选择自己喜欢的主题：

选择自己喜欢的主题后，点击复制其浏览路径：

打开自己创建的博客文件夹（我的就是MyBlog），右击打开Git Bash，输入命令:

git clone https://github.com/Suolawangzai/hexo-theme-nebula.git themes/nebula

上面的git clone后面输入的是选定主题的浏览地址；

执行完上述命令后，就将我们选定的主题nebula下载到Myblog目录下的themes文件夹下的nebula文件夹中；

然后打开站点配置文件，找到theme，将其默认主题样式theme:landscape修改为我们下载的主题theme:nebula；

修改完成后，再次部署网站，hexo g、hexo d（可以简写生成并上传：hexo d -g）查看网站的主题样式，有时候浏览器不会立刻显示出来，可以进行清除缓存hexo clean（我之前修改主题后，无论怎样部署，主题样式都不显示，后来换了个浏览器就可以了，本人也不太清除这个主题切换为什么有这样的问题）；
